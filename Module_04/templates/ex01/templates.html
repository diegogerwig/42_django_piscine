Django's Default Template Engine

Description

Django's default template engine is a powerful tool for rendering dynamic HTML pages. It allows for the separation of design from Python code, making templates more maintainable and easier to work with.

1. Blocks

Blocks are used for template inheritance, allowing child templates to override specific sections of parent templates.


{% verbatim %}
{% block content %}
    This is the default content
{% endblock %}
{% endverbatim %}

Child templates can override this block:


{% verbatim %}
{% extends "base.html" %}
{% block content %}
    This overrides the parent content
{% endblock %}
{% endverbatim %}

2. Loops (for ... in)

Loops allow iteration over lists or querysets in templates.


{% verbatim %}
{% for item in item_list %}
    <li>{{ item }}</li>
{% endfor %}
{% endverbatim %}

This will create a list item for each element in item_list.

3. If Control Structures

Conditional statements allow for content to be rendered based on certain conditions.


{% verbatim %}
{% if user.is_authenticated %}
    Hello, {{ user.username }}!
{% else %}
    Please log in.
{% endif %}
{% endverbatim %}

4. Display of Passing Context Variables

Context variables passed from views can be displayed directly in templates.


{% verbatim %}
<p>Welcome, {{ user_name }}!</p>
<p>Today is {{ current_date }}.</p>
{% endverbatim %}

Here, user_name and current_date are variables passed in the context dictionary from the view.

Conclusion

Django's template engine provides a flexible and powerful way to generate dynamic HTML content. By using blocks, loops, conditionals, and context variables, developers can create complex layouts while maintaining clean and readable code.
