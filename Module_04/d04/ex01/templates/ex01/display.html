Static Web Page Process

Description:

In this page, we describe the process causing the display of a static web page from a simple template going through a view, from the request to the answer:

  1. The user enters a URL in the browser, which sends a request to the server.
  2. The server receives the request and routes it to the appropriate Django view.
  3. The view function processes the request (in this case, it doesn't need to fetch data from a database).
  4. The view selects the appropriate template (in this case, our static page template).
  5. Django renders the template, replacing any template tags or variables with actual content.
  6. The rendered HTML is sent back to the user's browser as a response.
  7. The browser receives the HTML and any associated static files (like our style1.css).
  8. The browser renders the HTML and applies the CSS styles to display the page to the user.
Static Web Page Characteristics
1. In static web pages, Pages will remain same until someone changes it manually.
2. Static Web Pages are simple in terms of complexity.
3. In static web pages, Information are change rarely.
4. Static Web Page takes less time for loading than dynamic web page.
5. In Static Web Pages, database is not used.
6. Static web pages are written in languages such as: HTML, JavaScript, CSS, etc.
7. Static web pages does not contain any application program.
8. Static web pages require less work and cost in designing them.
